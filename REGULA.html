Lingua Programmatica

Veritas

The True Programming Language

A language designed to resemble clear, structured human English while remaining deterministic and easy to parse.

version 1.1.0  |  file extension .ver

§ 01

Philosophy

Veritas prioritizes readability over typing speed. Programs should read like well-structured English instructions. Every instruction must follow strict grammatical rules so that the compiler can translate the program into C code without ambiguity.

§ 02

File Extension

All Veritas source files must use the .ver file extension.

example.ver

§ 03

Sentence Termination

Every statement must end with a period (.). Multi-line statements are joined by the compiler and terminated by the final period.

Create 'x' as an int.

§ 04

Comments

Comments use C-style block syntax. Both inline and multi-line comments are supported. Comments are stripped before parsing.

/* This variable stores the result */
Create 'result' as a float with value 2.0.

§ 05

Identifiers

Variable names, function names, and array names must be enclosed in single quotes. This prevents ambiguity between English keywords and user-defined names.

'x'
'result'
'safeSqrt'
'values'

§ 06

String Literals

String literals must be enclosed in double quotes.

"Hello world\n"
"sqrt(%f) = %f\n"

§ 07

Library Inclusion

Use library or header interchangeably. The name is passed directly to #include <...>.

Include the library 'stdio.h'.
Include the library 'math.h'.
Include the header 'my_header.h'.

§ 08

Program Declaration

Every program must open and close with matching declaration statements using the same program name.

This is the program 'program_name'.
/* ... body ... */
End of the program 'program_name'.

§ 09

Type System

Veritas strips the English article (a or an) and passes the remainder directly to C as the type name. This means any valid C type can be used. Multi-word types such as double complex are fully supported.

VeritasC
a floatfloat
an intint
a doubledouble
a double complexdouble complex
a float pointerfloat*
a double complex pointerdouble complex*
nothingvoid
Rule of Article: Use an before vowel-initial types (an int, an unsigned int) and a before consonant-initial types (a float, a double). The compiler warns on mismatch but output is unaffected. nothing is the only reserved type keyword and always maps to void.

§ 10

Variable Creation

Variables may be declared with or without an initial value.

Create 'result' as a float.
Create 'count' as an int with value 1.
Create 'ptr' as a double pointer.
Create 'z' as a double complex.
Create 'PI' as a double with value 3.141592654.

§ 11

Array Creation

Array size must be a literal integer. Items are separated with commas; the final item is preceded by and. Multi-word types are supported.

Create 'values' as a float array of size 5 with values:
4.0,
9.0,
0.0,
-5.0,
and 16.0.

Create 'output' as a double complex array of size 8.

§ 12

Function Definition

Parameters are separated by commas or and. The return type follows returning. Use returning nothing for void functions.

Define the function 'safeSqrt'
with 'x' as a float,
'result' as a float pointer
returning nothing.

    /* body */

End function 'safeSqrt'.

Pointers in parameter lists render with the * adjacent to the name: float *result.

§ 13

Function Calls

Every call must end with stored to. Use stored to nothing to discard the return value.

Call 'sqrt' with 'x', stored to 'y'.
Call 'printf' with "Hello\n", stored to nothing.
Call 'dft' with
'input',
'output',
and 'N'
stored to nothing.

§ 16

Assignment

Replace 'x' with 0.0.
Replace value at 'result' with 2.1.   /* pointer dereference assignment */
Replace an element of 'arr' at index 'i' with 0.

§ 17

Conditionals

The closing End if repeats the original condition for readability. Otherwise is optional.

If 'x' is greater than 0.0:

    Replace 'y' with 1.0.

Otherwise:

    Replace 'y' with -1.0.

End if 'x' is greater than 0.0.

§ 19

Loops

The closing End iteration repeats the original bounds. Loop variables are declared as int automatically.

KeywordC condition
from A through Bi <= B
from A to Bi < B
 
For every iteration of 'i' from 0 through 4:

    /* body — i runs 0, 1, 2, 3, 4 */

End iteration of 'i' from 0 through 4.

For every iteration of 'k' from 0 to 'N':

    /* body — k runs 0 … N-1 */

End iteration of 'k' from 0 to 'N'.

§ 20 – 21

Arithmetic Operators

CVeritas
+plus
-minus
*multiplied by
/divided by

Bare Replace and Create ... with value statements support a single operator only. For compound expressions, use the quantity to create a parenthesised group.

Replace 'x' with 'a' plus 'b'.
/* x = a + b; */

Use the quantity to group sub-expressions, and semicolons to chain multiple groups into one compound expression.

Replace 'y' with the quantity 2 multiplied by 'x' plus 3; multiplied by the quantity 3 multiplied by 'x' plus 5.
/* y = (2 * x + 3) * (3 * x + 5); */

Nested quantities are supported — a the quantity group may appear as the RHS of an outer operator.

Replace an element of 'output' at index 'k' with
the quantity an element of 'output' at index 'k'
plus an element of 'input' at index 'n'
multiplied by the quantity 'cos_angle' plus 'sin_angle' multiplied by 'I'.
/* output[k] = (output[k] + input[n] * (cos_angle + sin_angle * I)); */

§ 22

Exponentiation

Exponentiation is not a built-in operator. Use pow from math.h.

Call 'pow' with 'x' and 2, stored to 'y'.

§ 23

§ 24

§ 25

Formatting Guidelines

Indentation is recommended for readability but not enforced. Multi-line statements are joined by the compiler as long as continuation lines do not begin with a reserved starter keyword. Nested structures such as loops and conditionals should be indented for clarity.

§ 26

Program Structure

A Veritas program has two logical sections. Definitions (Include, Create, Define) are placed at the top level and become globals or function definitions in the generated C. All other statements become execution statements inside main.

Veritas

Include the library 'stdio.h'.

Create 'x' as an int.

Define the function 'printVal'
with 'value' as an int
returning nothing.

    Call 'printf' with
    "Value = %d\n"
    and 'value',
    stored to nothing.

End function 'printVal'.

Call 'printVal' with 5,
stored to nothing.

Generated C

#include <stdio.h>

int x;

void printVal(int value)
{
    printf("Value = %d\n", value);
}

int main(void)
{
    printVal(5);
}

Full Example — example.ver

A complete program demonstrating function definition, pointers, conditionals, loops, and array access.

Include the library 'math.h'.
Define the function 'safeSqrt'
with 'x' as a float
and 'result' as a float pointer
returning nothing.

        Replace value at 'result' with 0.0.
    Otherwise:

        Call 'printf' with
        "Invalid input\n",
        stored to nothing.

        Call 'printf' with
