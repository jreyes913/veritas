Lingua Programmatica

Veritas

The True Programming Language

A language designed to resemble clear, structured human English while remaining deterministic and easy to parse.

file extension .ver

§ 01

Philosophy

Veritas prioritizes readability over typing speed. Programs should read like well-structured English instructions. Every instruction must follow strict grammatical rules so that the compiler can translate the program into C code without ambiguity.

§ 02

File Extension

All Veritas source files must use the .ver file extension.

example.ver

§ 03

Sentence Termination

Every statement must end with a period (.). Even structural constructs eventually terminate with a statement that ends in a period.

Create 'x' as an integer.

§ 04

Comments

Comments use C-style block syntax. Inline and multi-line comments are both allowed.

/* This variable stores the result */
Create 'result' as a float with value 2.0.

§ 05

Identifiers

Variable names, function names, and array names must be enclosed in single quotes. This prevents ambiguity between English keywords and identifiers.

'x'
'result'
'safeSqrt'
'values'

§ 06

String Literals

String literals must be enclosed in double quotes.

"Hello world\n"
"sqrt(%f) = %f\n"

§ 07

Library Inclusion

Include the library 'stdio.h'.
Include the library 'math.h'.
Include the header 'my_header.h'.

§ 08

Program Declaration

Every program must open and close with matching declaration statements.

This is the program 'program_name'.
/* ... body ... */
End of the program 'program_name'.

§ 09

Variable Creation & Type System

Veritas handles typing by stripping the English article (a or an) and passing the remainder directly to C. Multi-word types (like double complex) and pointers are supported.

Create 'result' as a float.
Create 'count' as an integer with value 1.
Create 'ptr' as a double pointer.
Create 'z' as a double complex.
Rule of Article: nothing maps to void. Pointers append pointer to the type name.

§ 10

Array Creation

Arrays may optionally be initialized with values. Items are separated with commas; the final item is preceded by and.

Create 'values' as a float array of size 5 with values:
4.0,
9.0,
0.0,
-5.0,
and 16.0.

§ 20 – 21

Arithmetic Operators

OperationSyntax
+plus
-minus
*multiplied by
/divided by

Arithmetic follows standard precedence rules. For complex formulas, use the quantity to group operations.

Replace 'x' with 2 multiplied by the quantity 3 plus 4.
/* Translated to: x = 2 * (3 + 4); */

Compound expressions can be chained with semicolons for grouping multiple parenthetical operations at once.

Replace 'result' with the quantity 'a' plus 1; divided by the quantity 'b' minus 1.
/* Translated to: result = (a + 1) / (b - 1); */

§ 22

Exponentiation

Exponentiation is not a built-in operator. Use pow from math.h.

Call 'pow' with 'x' and 2, stored to 'y'.

§ 25

Formatting Guidelines

Indentation is recommended for readability but not strictly required. Nested structures such as loops and conditionals should be indented.

§ 26

Program Structure

A Veritas program has two logical sections. Definitions (Include, Create, Define) are placed at the top level. All other statements are execution statements and are automatically placed inside main when translated to C.
